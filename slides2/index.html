Prečo nie všetko priamo do main

Spoločná verzia má ostať stabilná

Keď do nej píšu všetci naraz, ľahko si navzájom rozbijú prácu alebo tam pristane niečo nedokončené. Potrebujeme bezpečný priestor, kde si zmenu spravíme a doladíme.
Branch (vetva)

Vlastná bezpečná línia na zmenu

main = oficiálna verzia · vetva = moja paralelná línia, kde bezpečne robím
Dve vetvy, dva účely
main
stabilná spoločná verzia
do nej nepracujem priamo
feature vetva
moja pracovná línia
nové commity idú sem
Kým robím na vetve, main stojí v bezpečí.
Ako pomenovať vetvu

Názov = čo na vetve robím

add-about-page fix-header-typo update-contact-info
Krátko, výstižne, bez diakritiky a medzier.

Konflikt

Keď dvaja zmenia to isté miesto

Git nevie sám rozhodnúť, ktorá verzia je správna — tak sa spýta teba. Nie je to chyba, len rozhodnutie, ktoré musí spraviť človek.
Rôzne súbory git spojí sám. Konflikt = rovnaký riadok, dve verzie.
Ako konflikt vyzerá

Vyber správnu verziu

<<<<<<< tvoja verzia
Obľúbený nápoj: káva
=======
Obľúbený nápoj: čaj
>>>>>>> verzia kolegyne
Rozhodni → zmaž značky → nechaj správny obsah → ulož → commit.
Hands-on · 5

Vyrob a vyrieš konflikt

Dve z vás zmeňte ten istý riadok. Pri merge GitKraken nahlási konflikt → otvor súbor → vyber/skombinuj verziu → ulož → commit.
→ GitKraken: Resolve conflict (vo dvojiciach)
